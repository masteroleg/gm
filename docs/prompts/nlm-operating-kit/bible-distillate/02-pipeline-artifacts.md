This section covers Pipeline and Artifacts. Part 2 of 5 from BMAD Visual Storytelling Bible v4.

## 4 Communication Modes
1. **Self-read / leave-behind / pre-read** — primary mode of this playbook; read without speaker, forwarded, used as decision doc; requires: takeaway titles + summary-layer + appendix-layer + annotations + self-contained visual blocks + more context than live slides
2. **Speaker-support slides** — visual lives with verbal layer; less text, faster rhythm; secondary in this playbook
3. **Single-screen summary / infographic** — one-screen orientation / story-test before full deck / portable summary
4. **Report / memo / appendix-led brief** — when material is too analytical and text-dense for slides

## NLM Capability Map (Studio, web/desktop)
- Available artifacts: Notes, Audio Overviews, Video Overviews, Mind Maps, Reports, Data Tables, Flashcards/Quizzes, Slide Decks, Infographics
- Notebook architecture: each notebook independent; NLM does not work across multiple notebooks simultaneously; sources = static copies at import; notes used only when explicitly selected or converted to sources
- Exports: Reports → Docs; Data Tables → Sheets (table tab 1, citations tab 2); Notes → Docs or Sheets; exported file changes do NOT sync back; sharing permissions NOT auto-transferred
- Custom prompts: available for Reports, Audio/Video Overviews, Flashcards, Quizzes, Slide Decks, Infographics — use "View custom prompt" for reverse engineering
- Mobile limits: chat/audio/video/quizzes/infographics/slide decks available; notes/mind maps/reports/data tables NOT on mobile; full experience requires desktop
- Account/plan limits: always reference official Upgrade NotebookLM page — do not rely on memorized numbers

## Source Law
- If agent does not understand what actually imports into NLM → builds bad visual storytelling on false assumptions
- Static copy: NLM stores static copy at import; source is NOT live; must check freshness
- Websites: URL does NOT bring visual layer; NLM does NOT see website as designer sees it; add screenshots separately
- Google files: footnotes and comments NOT imported; critical caveats/legal notes/assumptions must be moved to explicit source or note
- YouTube: useful primarily as transcript; visual motifs/editing do NOT transfer; add screenshots/notes separately

### Source Compensation Rules (mandatory if preserving evidence + visual integrity)
1. Import images/diagrams/screenshots/deck references separately
2. Move footnotes/assumptions/comments to explicit notes
3. Update stale sources manually
4. Do NOT treat web source as full visual reference
5. Use notes-to-source as narrative + visual direction control

## Universal BMAD Pipeline (10 stages)
- **Stage 0. Environment Gate**: desktop/mobile; consumer/workspace; edit access; export/public share ability; available Studio artifacts
- **Stage 1. Reader Model**: who reads; prior knowledge; what to understand/decide; time budget; device context; whether forwarded; whether drill-down needed
- **Stage 2. Big Idea**: one phrase as conclusion
- **Stage 3. Horizontal Logic Strip**: write only slide/page titles first — if they do not tell the story alone, document not ready
- **Stage 4. Slide/Page Manifest**: per screen — purpose / takeaway / proof / visual function / visual form / text density / appendix candidate
- **Stage 5. Narrative Notes** (create in NLM, convert to sources): (1) Reader Brief; (2) Big Idea + Logic Strip; (3) Style/Mood/Anti-cliche Brief; optional: Evidence Notes / Comparative Criteria Note / Appendix Requirements Note
- **Stage 6. Source Curation**: remove irrelevant; check freshness; add missing evidence; add visual references + screenshots separately; select subset for specific artifact
- **Stage 7. Artifact Choreography**: do NOT jump straight to Slide Deck; determine which artifact chain best solves the task
- **Stage 8. Generate**: cheap understanding artifacts first (Mind Map / Report / Data Table / Infographic) → then high-commitment (Detailed Deck / Presenter Slides / Video Overview)
- **Stage 9. Grounded QA**: chat with citations / source spot-check / logic strip review / visual fit audit / accessibility gate / export-handoff gate
- **Stage 10. Revise or Rebuild**: apply Law 8

## Artifact Chains (by task type)
| Task | Recommended NLM Chain | Final Medium |
|---|---|---|
| Discovery/orientation | Fast Research → Sources → Mind Map → Report → Infographic | one-screen summary / discovery brief |
| Diagnosis | Sources → Report → Data Table → Evidence Deck | self-read deck / appendix-led brief |
| Strategy | Sources → Report → Data Table → Comparison Deck | self-read decision deck |
| Sales/persuasion | Sources → Reader Brief Note → Infographic → Detailed Deck → Revise | self-read proposal / leave-behind |
| Training/SOP | Sources → Report → Data Table → Process Infographic → Detailed Deck | training packet |
| Technical explanation | Sources → Mind Map → Architecture Report → Annotated Visual Deck | self-read explainer |
| Executive update | Sources → Briefing Report → Executive Summary Deck → Appendix | executive pre-read |
| Research synthesis | Sources → Mind Map → Report → Video Overview → Detailed Deck | synthesis deck / memo |

## When to Use Each Artifact
- **Mind Map**: see field structure; check missing subtopics; find natural branches; set architecture summary before deck
- **Report**: readable synthesis; FAQ/briefing/study guide; text-first logic check; raw material for later deck
- **Data Table**: explicit comparison; repeated structures; comparison slide scaffold; separate comparison logic from narrative
- **Infographic**: quick story spine test; one-screen orientation; narrative overload check; social/portable summary
- **Detailed Deck**: send for independent reading; combine summary + evidence + recommendation; support async decision
- **Video Overview**: socialize topic structure quickly; high-level walkthrough; pre-brief before detailed artifact — NOT a substitute for self-read evidence document
