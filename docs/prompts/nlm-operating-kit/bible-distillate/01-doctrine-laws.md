This section covers Doctrine and Laws. Part 1 of 5 from BMAD Visual Storytelling Bible v4.

## Doctrine
- Agent task = designing a managed visual path to understanding — not text + packaging
- Correct mental model: recipient state change → semantic steps → visual operations → medium → NLM pipeline
- Weak agent: gather facts → organize sections → format nicely
- Strong agent: (1) what must change in reader; (2) semantic steps to cause that change; (3) cognitive job of visual at each step; (4) choose medium and NLM pipeline
- If agent cannot answer "what exactly does the visual do at this step?" → work not started

## 4 Criteria for Strong Output (all 4 simultaneous)
1. Big Idea + clear part logic
2. Clear reading order (what to see first, second, third)
3. Visual does cognitive work (explains / compares / simplifies / proves / guides) — not decorates
4. Medium matches usage mode (self-read ≠ stage deck)

## 3 Mandatory Questions (answer before any task)
1. What must change in the reader after contact with the material?
2. What semantic steps cause that change?
3. What exactly must the visual do at each step?
- Q3 unanswered → work not started

## 8 Laws (non-negotiable)

### Law 1. Big Idea
- One main conclusion formulated as a judgment, not a topic
- BAD: "Про платформу eАкциз" / "Поточний стан системи" / "Проблематика"
- GOOD: "Поточний процес створює два системних вузьких місця, і обидва знімаються через X і Y"
- GOOD: "genu.mark ізолює виробничу лінію — це знижує ризик запуску без перебудови ERP"

### Law 2. One Slide / One Job
- Each slide/page does exactly one cognitive task: compares / shows process / flags risk / demonstrates structure / proves claim / materializes abstraction / leads to action
- If one screen does 5 jobs → split it

### Law 3. Takeaway Title
- Title = first line of meaning, not a label or divider
- FORBIDDEN: "Ринок" / "Проблема" / "Висновки" / "Чи варто нам...?"
- REQUIRED: "Зростання утримання йде тільки в одному сегменті" / "genu.mark не керує ERP — воно керує лінією"

### Law 4. Visual Fit
- Visual must match the cognitive operation of the slide
- comparison → comparison form; process → process form; evidence → evidence form; recommendation → recommendation form
- Wrong form = screen is defective even with good style

### Law 5. Information-carrying Imagery
- Every image must perform a cognitive function: explain / localize problem / strengthen proof / materialize abstraction / support comparison / accelerate orientation
- If agent cannot prove what cognitive work the image does → delete it

### Law 6. Summary + Appendix
- Self-read materials require two layers: summary-layer (fast route) + appendix-layer (drill-down / proof depth)
- Stage deck does not require this; self-read always does

### Law 7. Evidence Integrity
- Key claims must have: source + context + explanatory text + methodology caveat when needed
- Never assume "data speaks for itself"

### Law 8. Rebuild vs Revise vs Partial Rebuild
- REBUILD when: thought changes / audience changes / architecture changes / source set changes / argument order changes / number of semantic blocks changes
- REVISE when: local wording / layout / visual emphasis / readability / specific visual on specific slide
- PARTIAL REBUILD when: tone/framing changes but structure stays — rebuild narrative notes (Reader Brief, Big Idea, Style note) with new tone; reuse existing visual direction (extract via "View custom prompt"); re-generate from updated sources
- Using Revise to fix a changed argument = structural error; it will produce cosmetic patches on wrong logic
- Example partial rebuild: Executive Deck structure is correct but tone is "too architectural, not enough selling energy" → rebuild Reader Brief with tone=sell, rewrite Big Idea as reader outcome, keep the same slide structure and visual style
