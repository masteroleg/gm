## Executive Summary

**genu.im** — платформа верификации подлинности продуктов, построенная поверх украинской государственной системы eАкциз (xtrace.gov.ua). Работает как интегратор: не конкурирует с реестром, а добавляет ценность — управление кодами на производственной линии, аналитика для брендов и UX верификации для потребителей.

**Бизнес-модель:** usage-based B2B (per-code-unit), без капитальных затрат для клиентов.
**Инфраструктура:** Azure serverless (Storage Tables/Blob/Queues + Service Bus Basic) — ~$2/мес при текущей нагрузке, линейно масштабируется.
**Текущий масштаб:** 25M+ промаркированных единиц.

## Step 2: Vision Discovery

### Регуляторный контекст

**eАкциз** (электронный акциз) — украинский государственный мандат на маркировку продукции. Государственный реестр: **xtrace.gov.ua**.

- Текущий охват: алкоголь, табак
- Статус: в разработке / пре-запуск (2026)
- Потенциальное расширение: пищевая промышленность, косметика

genu.im позиционируется как **интегратор над xtrace.gov.ua** — не альтернатива, а дополнительный слой. Правительство видит платформу как партнёра по внедрению мандата.

### Продуктовые треки

#### 1. genu.mark (sub-brand) — Regulatory Compliance Layer
- Управление кодами маркировки на производственной линии
- Интеграция с оборудованием: Domino, Markem-Imaje, Omron
- Передача данных в xtrace.gov.ua
- **Ключевой актив:** данные производственной линии принадлежат genu.im независимо от государственного реестра
- Вендоры оборудования пробовали войти на рынок со своими решениями — не удалось (война + стоимость)

#### 2. Brand Protection & Analytics
- Трек качества потребительского UX (completion rate, скорость, повторные сканирования)
- Аналитика по продукту, географии, временным паттернам
- ROI для брендов: данные о подлинности = доверие потребителей = повторные покупки
- Страница верификации как touchpoint бренда (не просто "authentic/not")

#### 3. genu.im Consumer Verification — UX Layer
- Концепция "Case File → Verdict": 4 состояния — AUTHENTIC / SUSPICIOUS / NOT FOUND / OFFLINE
- Target: верификация за ≤2 секунды
- Consumer-brand data loop: качество UX → completion rate → богатство данных → ROI бренда → инвестиции в платформу
- Страница верификации как инструмент доверия к самой платформе (не только к продукту)

### Финансовая модель (скорректировано после Reverse Engineering)

| Показатель | Значение |
|------------|----------|
| **Реалистичный ARR (2–3 года)** | **$2–12M/год** |
| Целевой сегмент | 400–2,400 B2B клиентов |
| ARPU | $5,000–$20,000/год |
| Модель | Usage-based (per-code-unit) + минимальный контракт |
| Драйвер роста | Регуляторный мандат еАкциз (forced demand, CAC → 0) |

> **Примечание:** Цифра $524M была ранее озвучена как aspirational/pitch-deck ориентир. Операционная модель на 2–3 года: $2–12M ARR при 400–2,400 клиентах.

**Юнит-экономика (ориентир):**
- Print: ~$2/1,000 кодов
- Storage: ~$0.43/1,000 кодов
- Verification: ~$7/1,000 кодов
- Azure при 100K клиентах: ~$50–150K/мес → <0.3% от ARR на том уровне

**GTM канал:** Партнёрский (бухгалтеры, налоговые консультанты, вендоры оборудования) + self-serve onboarding при регуляторном давлении.

### Ключевые дифференциаторы

1. **Производственные данные** — слой данных, которым genu.im владеет независимо от государственного реестра
2. **Регуляторная интеграция** — позиция интегратора, не конкурента (MoU с Минагрополитики в разработке)
3. **Барьер вхождения подтверждён** — вендоры оборудования (Domino, Markem-Imaje) не смогли войти на рынок
4. **Consumer loop** — UX данных потребителя создаёт ценность для брендов, финансируя улучшение платформы

### Инсайты из Advanced Elicitation

**First Principles (метод 3):**
genu.im — не просто "SaaS для маркировки". Фундаментальное позиционирование: **lifecycle operator of product identifier** — платформа, которая управляет жизнью идентификатора от производственной линии до потребительского сканирования.

**Stakeholder Round Table (метод 2):**
- *Dmytro (инвестор):* Unit economics убедительны, ключевой риск — регуляторный таймлайн
- *Yaroslav (compliance):* SLA 99.9%, поддержка offline-режима критична для производства
- *Mykhailo (гос. партнёр):* MoU как путь, не конкуренция с xtrace.gov.ua
- *Oksana (бренд):* Аналитика completion rate = ROI обоснование для маркетинга
- *Tetyana (потребитель):* ≤2 секунды или доверие теряется

**Reverse Engineering (метод 4):**
$524M aspirational цель не сходится с реальным B2B рынком Украины. Рабочая операционная цель: **$2–12M ARR** на горизонте 2–3 лет при реалистичном охвате 400–2,400 регулируемых производителей с ARPU $5–20K.

## Step 4: Market & Competitive Analysis

### Конкурентная карта

| Игрок | Тип | Маркировка на линии | Аналитика для брендов | Consumer verification | Ценообразование |
|-------|-----|--------------------|-----------------------|-----------------------|-----------------|
| **xtrace.gov.ua** | Государственный реестр | ❌ | ❌ | ❌ | Бесплатно (мандат) |
| **Сайтекс** | Прямой конкурент | ✅ | ✅ | ❌ (гипотеза) | $250–500/мес/линия |
| **genu.im** | Претендент | ✅ | ✅ | ✅ | Usage-based per code |
| Дешёвые китайские принтеры + файл | DIY | Частично | ❌ | ❌ | Capex только |

### Сайтекс — детальный профиль

**Позиционирование:** "Друг xtrace.gov.ua" — интегратор над государственным реестром (аналогичная позиция с genu.im).

**Что делает:**
- Управление печатью на производственной линии (интеграция с принтерами)
- Аналитика для брендов

**Что не делает (гипотеза):**
- Consumer-facing страница верификации (покупатель сканирует QR → получает вердикт)

**Ценообразование:**
- ~$250/мес за линию (до определённого количества кодов)
- ~$500/мес за линию при бо́льшем объёме
- Модель: фиксированная подписка per-line, не per-code

**Сравнение моделей ценообразования:**

| | Сайтекс | genu.im |
|---|---|---|
| Структура | Фикс/линия/мес | Usage-based per code |
| Выгодно при | Высоком объёме на линии | Нерегулярном или растущем объёме |
| Риск для клиента | Платит даже в простой | Платит только за факт |
| Предсказуемость | Высокая | Средняя |

### Текущее поведение рынка

**Большинство производителей сейчас:**
- Не используют уникальные коды вообще, или
- Используют дешёвые китайские принтеры + печатают из файла (не интеграция с реестром)

**Импликация:** Это не рынок замены вендора — это рынок **смены парадигмы** (статические/отсутствующие коды → уникальная маркировка с верификацией). Продажа сложнее, но:
- Барьер копирования выше (нет укоренившейся привычки к конкуренту)
- Регуляторный мандат eАкциз создаёт принудительный переход — производитель не выбирает "менять или нет", только "с кем"

### Дифференциация genu.im от Сайтекс

**Единственный уникальный слой genu.im:** Consumer verification — страница, которую видит покупатель после сканирования.

Это не просто UX-фича. Это:
1. **Замыкает петлю данных:** сканирование потребителя → данные о географии/времени/частоте → ценность для бренда
2. **Создаёт сетевой эффект:** чем больше потребителей сканируют → богаче данные → выше ROI для бренда → бренд платит больше
3. **Барьер для Сайтекс:** добавить consumer layer — это другой продукт, другая команда, другой фокус

### Ключевые выводы

1. **Сайтекс = ближайший конкурент**, делает 2/3 треков genu.im, работает на том же рынке
2. **Consumer verification = единственный явный дифференциатор** на сегодня
3. **Производственные данные = скрытый дифференциатор** — кто владеет данными линии, тот не зависит от реестра
4. **Ценовая модель genu.im (per-code)** выгоднее для малого/среднего производителя с нерегулярным объёмом; Сайтекс выгоднее для крупных стабильных линий
5. **Главный риск:** Сайтекс добавляет consumer layer — тогда дифференциация только в данных производственной линии и UX качестве

## Step 5: Business Model & Roadmap

### Текущее состояние продукта

**MVP статус:** Все три трека работали в production для Укравит (средства защиты растений).

| Трек | Статус |
|------|--------|
| genu.mark — маркировка на линии | ✅ Работало для Укравит |
| Brand Protection & Analytics | ✅ Работало для Укравит |
| Consumer Verification (страница) | ✅ Работало для Укравит |

**Ключевой инсайт:** Первый реальный деплой был **добровольным** (Укравит сам захотел), не regulatory-driven. Это означает, что ценностное предложение работает вне eАкциз-мандата — brand protection как самостоятельный мотиватор.

**Почему нужно переделывать:** MVP накопил технический долг и архитектурные решения первого прохода. Нужен рефакторинг с учётом полученного опыта перед масштабированием.

### Первый клиент — Укравит

| | |
|---|---|
| **Компания** | Укравит |
| **Вертикаль** | Средства защиты растений |
| **Драйвер** | Добровольный (бренд-защита, не eАкциз) |
| **Статус** | Проект заморожен |
| **Урок** | Добровольный клиент без регуляторного давления = высокий риск заморозки. Следующий клиент должен иметь compliance-мотивацию |

### Приоритеты на ближайшие 3 месяца

1. **Production-ready платформа** — рефакторинг MVP с учётом опыта Укравит
2. **Первый платящий клиент** — из алко/табак вертикали (eАкциз-давление как страховка от заморозки)

> Государственное партнёрство / MoU — не в приоритете на текущем этапе.

### Бизнес-модель

**Тип:** Usage-based B2B — три отдельных компонента, оплата за факт использования.

#### Действующая модель ценообразования (подтверждена данными января 2026)

> ⚠️ Это НЕ единая "цена за код" — три отдельных события с отдельными ценами.

Consumer verification структурно не работает в этой вертикали. Верификационный трек (и $7/1K выручка от него) реализуется только в **consumer-facing продуктах**: алкоголь, еда, косметика.

#### Сравнение с конкурентом

| | Сайтекс | genu.im |
|---|---|---|
| Структура | Фикс $250–500/мес/линия | Pay-per-use (3 компонента) |

**Вывод по ценообразованию:** модель конкурентна на малых объёмах; на крупных объёмах премия оправдана дополнительными треками (аналитика + consumer verification), которых нет у Сайтекса.
