## What This Kit Does

This is a controlled `sales-kit source system` for regulated `еАкциз` motion.

Its job is to:
- sell `genu.mark` first;
- keep `ERP` and `еАкциз` visibly external;
- give Notebook LM a compact, repeatable, source-grounded generation path;
- keep buyer-facing outputs modern, Ukrainian, and ready to forward without oral decoding.

## Non-Negotiable Strategy

- main object of sale in regulated motion = `genu.mark`;
- `genu.mark` = strong line core;
- `ERP` = external contour in two roles:
  - `ERP -> source of production job`;
  - `ERP / internal systems <- exchange of results and statuses`;
- `еАкциз` = separate external state contour;
- `Dominanta` = Domino equipment + trusted commercial entry point;
- `Logictime` = architecture, integrations, exceptions, implementation, launch;
- `genu.im` = supporting proof context only when needed;
- `genu.code` = separate voluntary branch, never early regulated narrative.

## Compact Upload Rule

Default upload pack for mandatory regulated generation:
1. `regulated-final-bundle/01-control-packet.md`
2. `regulated-final-bundle/02-approved-regulated-source-packet.md`
3. one approved buyer-facing regulated source asset;

Do not upload prompt docs or QA docs as Notebook LM sources.

## Hard Fail Conditions

Reject any regulated output immediately if:
- it sells a platform instead of `genu.mark`;
- it shows `ERP` as part of the core;
- it shows `еАкциз` as part of the core;
- it introduces `genu.code` in early regulated narrative;
- it mixes regulated and voluntary logic;
- it sounds like operator-source draft;
- it does not move to audit / architecture session / pilot.
