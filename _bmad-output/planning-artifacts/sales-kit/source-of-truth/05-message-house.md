# Message House

## Core promise

`genu.mark helps implement еАкциз without line chaos because it gives the buyer a strong core first and connects external contours deliberately.`

## Narrative order

Use this order in regulated materials:
1. `еАкциз` implementation is inevitable;
2. the buyer's real question is how to launch without production risk;
3. `genu.mark` is the strong line core;
4. architecture = core plus external contours;
5. risk lives at the interfaces and in exceptions;
6. strong core first reduces overload and manual error;
7. `Dominanta + Logictime` is the safer path;
8. next step = audit / architecture session / pilot.

## Value formula order

When explaining value, keep this order:
1. `genu.mark`
2. `Domino / Dominanta`
3. `Logictime implementation discipline`
4. `genu.im` only if supporting proof is needed.

## Safe short lines

- `genu.mark is the strong line core for еАкциз implementation.`
- `ERP is external in two roles: source of production job and exchange of results/statuses.`
- `еАкциз is a separate external contour.`
- `Strong core first reduces launch risk.`
- `Dominanta brings reliable Domino equipment; Logictime designs and launches the contour around the core.`

## Supporting context

Only after the core message lands may the story mention:
- evidence and reviewability;
- public proof context;
- later supporting context from `genu.im`.

`genu.code` does not belong in early regulated narrative.

## End-state rule

Mandatory regulated assets must end in one of these moves:
- audit;
- architecture session;
- pilot.

## Editorial test

Reject the copy if it:
- does not clearly sell `genu.mark`;
- hides the ERP split;
- blurs `еАкциз` into the core;
- sounds like draft notes;
- ends without a concrete next step.
