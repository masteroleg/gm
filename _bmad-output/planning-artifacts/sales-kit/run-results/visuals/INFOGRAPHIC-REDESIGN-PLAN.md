## Затверджений візуальний стиль

**Джерело:** `notebooklm-source/04-visual-output-rules.md`

| Параметр | Значення |
|----------|----------|
| **Стиль** | Regulated industrial premium |
| **Мова** | Українська (усі buyer-facing тексти) |
| **Палітра** | Graphite, warm white, deep blue/teal, один signal color |
| **Типографіка** | Одна font pairing, великі headline, короткі блоки |

- English headings/body у buyer-facing матеріалах
- Glassmorphism, neon, startup gradients
- Декоративні 3D-ефекти
- Stock-photo контент
- Dashboard-like UI
- "Risk-free", "гарантований", "під ключ" без доказів

## Текстовий контент

### Заголовок (обрати один)

1. `"Для еАкциз треба продавати готовий шлях, а не тільки hardware"`
2. `"Одна задача, а не чотири фрагменти"`
3. `"Domino equipment + genu.mark = сильніша відповідь для клієнта"`

### Підзаголовок (обрати один)

1. `"Коли клієнт питає про еАкциз, розмова вже більша за принтер."`
2. `"Обладнання важливе, але клієнт купує спокійніший шлях запуску."`

### Ліва сторона (фрагментована)

```
Обладнання саме по собі не закриває ядро лінії
і не знімає ризик стиків із зовнішніми контурами.
```

### Права сторона (керований шлях)

```
Domino equipment плюс genu.mark дають сильніше ядро запуску,
а ERP і еАкциз підключаються окремо й усвідомлено.
```

### 3 результати

1. `Менше координаційного ризику`
2. `Зрозуміліша відповідальність`
3. `Простіший наступний крок`

### CTA

```
Наступний крок: спільний аудит / архітектурна сесія
```
