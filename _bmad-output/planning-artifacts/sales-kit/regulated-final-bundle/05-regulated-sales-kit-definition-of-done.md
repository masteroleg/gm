# Regulated Sales Kit Definition of Done

## What is sold

- Sell `genu.mark`.
- Sell safe regulated implementation.
- Sell `Dominanta + Logictime` as the safer route to rollout.

## What is core

- `genu.mark` is the line core.
- The core contains line flow, verification, reject / rework control, aggregation, and line result state.

## What is external

- `ERP -> source of production job`.
- `ERP / internal systems <- exchange of results and statuses`.
- `еАкциз` as a separate external contour.

## What cannot be mixed

- regulated and voluntary logic;
- `genu.code` in the early regulated story;
- `genu.im` as the opening sell;
- platform-first and ecosystem-first framing.

## What CTA must appear

At least one of these must be explicit:
- audit;
- architecture session;
- pilot.

## What fails QA instantly

- selling a platform instead of `genu.mark`;
- showing `ERP` or `еАкциз` as part of the core;
- hiding the ERP split behind vague `ERP integration` wording;
- buyer-facing output that sounds like an internal draft;
- missing next step.

## Release condition

The kit is done only when all six mandatory regulated assets have a matching brief, prompt, runbook, and QA file, and each asset can be generated and released without verbal correction.
