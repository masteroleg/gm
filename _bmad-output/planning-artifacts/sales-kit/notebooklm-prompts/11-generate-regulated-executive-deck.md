Use only the uploaded sources. Do not invent architecture, benefits, claims, or steps that are not supported by those sources.

- Sell `genu.mark` first.
- Treat `genu.mark` as the line core.
- Treat `ERP` as external in two explicit roles:
  - `ERP -> source of production job`
  - `ERP / internal systems <- exchange of results and statuses`

## Output contract

Return 8 slides unless the sources clearly justify splitting one section into 9 or 10. Use this format for each slide: `Slide N - title`, `Key message`, `3-5 bullets`, `visual note`, `speaker line`.

Keep slide numbering clean. Do not insert broken references, footnote numbers, placeholders, or artifacts like `1`, `more_horiz`, or source markers inside the text.

- it sells a platform instead of `genu.mark`;
- it hides the ERP split under vague integration language;
- it shows `ERP` or `еАкциз` as core;
- it skips interface and exception risk;
- it changes the required 8-slide order without a good source-grounded reason;
